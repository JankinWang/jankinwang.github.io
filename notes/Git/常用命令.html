# 提交

git commit -m '第一行提交原因'  -m '第二行提交原因'

## 只提交 `修改` 或 `删除`， `新增`的文件不会提交
git commit -am '提交原因'

## 修改最新一条提交记录的提交原因
git commit --amend -m '提交原因'

# 远程仓库

# 推送

## git push <远程主机名> <本地分支名>:<远程分支名>
git push origin master
git push origin dev:master
git push origin dev^:master

## 会删除远程仓库中的dev分支
git push origin :master

## 如果指定一个远程没有的分支（newBranch），则会在远程自动创建分支
git push origin master:newBranch

# 克隆

git clone [仓库URL]
git -b [分支名] [仓库URL]

# 拉取

## pull 相当于 fetch + merge
git pull ## 默认远程仓库的默认分支
git pull origin master
git pull origin foo

## 它先在本地创建了一个叫 foo 的分支，
## 从远程仓库中的 master 分支中下载提交记录，
## 并合并到 foo，然后再 merge 到我们的当前检出的分支 bar上。
git pull origin foo:master

## 会创建一个本地分支dev
git fetch origin :dev

# 获取

## git fetch <远程主机名> <远程分支名>:<本地分支名>
git fetch
git fetch origin master
git fetch origin master:dev

## 会创建一个本地分支dev
git fetch origin :dev

# 远程追踪

## 手动创建一个远程分支的追踪
git checkout -b dev origin/main
git branch -u o/main dev

# 取消跟踪

## 删除readme1.txt的跟踪，并保留在本地。
git rm --cached readme1.txt

## 删除readme1.txt的跟踪，并且删除本地文件。
git rm --f readme1.txt

然后git commit即可。然后添加到 .gitignore 文件

# 提交日志

git log

git reflog

# 移动或重命名文件、目录

git mv a.md b.md -f
## 将 a.md 重命名为 b.md ，同时添加变动到暂存区，
## 加 -f 参数可以强制重命名，
## 相比用 mv a.md b.md 命令省去了 git add 操作

# 分支

# 管理分支

git branch <分支名> ## 创建分支

git branch   ## 列出所有分支
git branch -a ## 本地和远程版本库分支表
git branch -r ## 远程分支
git branch -vv ## 带有最后提交 id 和 提交信息的本地分支列表
git branch -d <分支名> ## 删除分支

## 强制修改分支位置（将分支指向其他提交记录）
git branch -f main HEAD~3
git branch -f main <哈希值>

# 切换分支

git checkout <分支名><提交记录哈希值> ## 切换分支
git checkout -b <分支名> ## 创建并切换到分支

## 相对引用
git checkout master^  ## 向上移动 1 个提交记录
git checkout master~<n> ## 向上移动n个提交记录，如 ~3
## 当一个提交是由两个父提交合并而来的，^ 默认会返回第一个父提交，^2 则会选择第二个父提交
git checkout master^2 ## 选择父提交记录
## 相对引用的链式操作
git checkout HEAD~^2~3

## 新语法
git switch

从本地版本库的 HEAD（也可以是提交ID、分支名、Tag名） 历史中检出 demo.html 覆盖当前工作区的文件，如果省略 HEAD 则是从暂存区检出

git checkout HEAD demo.html

这个命令会创建一个全新的，完全没有历史记录的新分支，但当前源分支上所有的最新文件都还在，真是强迫症患者的福音，但这个新分支必须做一次 git commit操作后才会真正成为一个新分支。

git checkout --orphan new_branch

# 合并分支

## 将其它分支合并到当前分支
git merge -e --no-ff <分支名>
# -e 编辑合并信息
## --no-ff 非 快进式合并

## 将当前提交的副本放在指定提交的后面
git rebase main
git rebase main feature
git rebase -i HEAD~3 ## 交互式 rebase

# 贮藏

git stash list ## 已贮藏
git stash show stash@{0} ## 展示一条记录
git stash drop stash@{0} ## 移除一条记录

git stash ## 贮藏未提交的文件

git stash pop ## 弹出最新

git stash apply stash@{0} ## 应用指定贮藏

# 标签Tag

## 创建 tag
git tag v1.0.0 <哈希值>|<分支名>|<HEAD>

## 距离当前提交最近的 tag
## <ref> 可以是任何能被 Git 识别成提交记录的引用
git describe <ref>

# 撤销变更

# reset

## 适合本地分支
git reset HEAD^ ## 直接将分支移动到上一条提交记录，并将当前记录的修改吐回工作区

# revert

## 适合远程分支
git revert HEAD ## 创建一个不包含当前提交记录修改的 新提交记录，以此达到撤销的效果

# 整理提交记录

## 将一些提交复制到当前所在的位置（HEAD）后面。 
## 如：将其他分支的某几个提交，依次复制到main(HEAD)后面
git cherry-pick <提交号>...
