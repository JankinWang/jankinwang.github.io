# 基本编码规范

# 命名规范

类名: 大驼峰

方法: 小驼峰 （包括函数）

常量: 全大写, 下划线分隔

变量: 全小写, 下划线分隔

# 代码风格

  • 类、方法 的 花括号 应另起一行

  • 控制结构的关键字后 必须 要有一个空格符，而调用方法或函数时则 不可 有。

  • 纯 PHP 代码文件 必须 省略最后的 ?> 结束标签。

  • 方法名称后 不可 有空格符

# 布尔型变量命名

是不是：is_____

能不能：can_

能不能怎么样: _able 如 Clickable 可点击的，是否可以点击

有没有：has_

非属性的成员变量前加 m 前缀，如 mClickable （POJO 类除外，不加 ）

静态的就最好加上 s 前缀（POJO 类除外，不加 ）

# 安全编程

用户输入的的数据必须【转码】，必须【验证数据格式】

复杂逻辑应该进行【错误处理】和 【异常处理】

数据库操作 必须对条件参数进行预处理，避免 SQL 注入
